## git commit 규칙

```text
[type]: 제목

[본문]

```

1. 제목, 본문 분리하기(한 줄 띄워서)
2. 제목 20자 이내로(제목 . 금지)
3. 본문은 제목에 대한 설명(무슨 기능을 하는지, 왜 만들었는지, 왜 작성을 했는지)

### type
feat: 새로운 기능 추가
fix: 버그 수정
docs: 문서 수정(API)
refact: 코드 리팩토링
test: 테스트 코드

### 제목
제목으로 최대한 파악이 가능하도록 작성

### 본문
제목에서의 상세 내용을 적기

## Running the app

```bash
# development
$ npm run start

# watch mode
$ npm run start:dev

# production mode
$ npm run start:prod
```

## Test

```bash
# unit tests
$ npm run test

# e2e tests
$ npm run test:e2e

# test coverage
$ npm run test:cov
```
